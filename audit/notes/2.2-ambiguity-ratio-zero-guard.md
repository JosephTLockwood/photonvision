## Direction of the ratio

All three ports compute:

    ambiguity = errors[0] / errors[1]   // best / alt

Per `PnpResult.java` line 30-31 Javadoc: "the ambiguity value represents the ratio of
reprojection error in the best solution to the alternate **(best / alternate)**".

OpenCV's `solvePnPGeneric` documentation guarantees solutions sorted ascending by
reprojection error, so `errors[0] ≤ errors[1]` → `ambiguity ∈ [0, 1]`. **Lower =
more confident** (best fits much better than alt). The producer-side semantics match the
consumer-side selection at
[PhotonPoseEstimator.java:488](photon-lib/src/main/java/org/photonvision/PhotonPoseEstimator.java#L488):
`if (targetPoseAmbiguity != -1 && targetPoseAmbiguity < lowestAmbiguityScore)` — uses `<`,
lower wins. Consistent. ✓

## Denominator-zero / NaN guard — missing on all three ports

Java [OpenCVHelp.java:582](photon-targeting/src/main/java/org/photonvision/estimation/OpenCVHelp.java#L582):

    return Optional.of(new PnpResult(best, alt, errors[0] / errors[1], errors[0], errors[1]));

C++ [OpenCVHelp.h:266](photon-targeting/src/main/native/include/photon/estimation/OpenCVHelp.h#L266):

    result.ambiguity = errors[0] / errors[1];

Python [openCVHelp.py:258](photon-lib/py/photonlibpy/estimation/openCVHelp.py#L258):

    ambiguity=reprojectionError[0, 0] / reprojectionError[1, 0],

No port checks `errors[1] == 0` or `isnan(errors[1])` before division. All three rely on
IEEE-754 semantics — `x / 0` → `±Inf` (or `NaN` if x is also 0), `x / NaN` → `NaN`.

## NaN-retry logic only guards errors[0]

The retry block at
[OpenCVHelp.java:568-575](photon-targeting/src/main/java/org/photonvision/estimation/OpenCVHelp.java#L568-L575):

    if (!Double.isNaN(errors[0])) break;
    else { add noise; retry; }
    // ...
    if (Double.isNaN(errors[0])) throw new Exception("SolvePNP_SQUARE NaN result");

Checks **only** `errors[0]`. If `errors[0]` is valid but `errors[1]` is NaN, the retry
exits, the NaN guard passes, and `ambiguity = valid / NaN = NaN` is returned to the
caller.

## Downstream impact: NaN poisons `AVERAGE_BEST_TARGETS`

`LOWEST_AMBIGUITY` strategy at
[PhotonPoseEstimator.java:488](photon-lib/src/main/java/org/photonvision/PhotonPoseEstimator.java#L488):

- `NaN < x` evaluates to `false` (always). → tag is silently excluded. Safe.
- `+Inf < x` (e.g. from `errors[1] == 0`) → also `false`. → tag excluded. Safe.

`AVERAGE_BEST_TARGETS` at
[PhotonPoseEstimator.java:700-739](photon-lib/src/main/java/org/photonvision/PhotonPoseEstimator.java#L700-L739):

    if (targetPoseAmbiguity == 0) { return immediately; }        // NaN == 0 → false
    totalAmbiguity += 1.0 / target.getPoseAmbiguity();           // 1/NaN = NaN
    // ...
    double weight = (1.0 / amb) / totalAmbiguity;                // NaN/NaN = NaN
    transform = transform.plus(estimatedPose.getTranslation().times(weight));  // NaN translation
    rotation  = rotation.rotateBy(estimatedPose.getRotation().times(weight));  // NaN rotation
    return new EstimatedRobotPose(new Pose3d(transform, rotation), ...);       // NaN pose

A single tag with NaN ambiguity **poisons the entire weighted average**, producing an
`EstimatedRobotPose` with NaN translation and rotation. Whether wpilib's
`addVisionMeasurement` filters non-finite measurements is the user's downstream defense.

`+Inf` is handled gracefully: `1.0 / Inf = 0`, so the tag contributes 0 weight to the
average. Safe.

## Practical likelihood

`errors[1] = NaN` while `errors[0]` is finite is **very unusual** out of OpenCV's
`solvePnPGeneric(SOLVEPNP_IPPE_SQUARE)`. Both IPPE solutions share the same input
correspondences; degeneracy usually affects both. But not always — OpenCV's IPPE solver
applies separate post-checks per solution, and contrived synthetic inputs could trigger
single-solution NaN. The retry-with-noise logic at lines 569-575 exists because the
authors already observed OpenCV returning NaN; the gap is that they only retried on
`errors[0]` NaN.

`errors[1] = 0` (Inf ambiguity) — also rare. Would need a synthetic input that perfectly
fits the alternate solution.
